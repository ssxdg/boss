# Boss 直聘自动投递浏览器插件需求文档

## 1. 项目目标

在 `E:\LLM\pubiln` 目录下创建一个 Chrome/Edge 浏览器插件项目，用于在 Boss 直聘网页版中，根据用户配置的职位筛选条件和沟通模板，自动浏览职位、筛选职位、发送打招呼内容并记录投递结果。

项目技术栈：

- 浏览器扩展：Chrome/Edge Manifest V3
- 前端：React + TypeScript
- 构建：优先使用 Vite 或等价脚手架创建工程
- 存储：`chrome.storage.local`
- 数据导入导出：JSON 文件

创建项目时必须使用终端脚手架初始化工程，不要逐个手工创建基础项目文件。

## 2. 合规与安全边界

插件只在用户已登录 Boss 直聘网页版后运行，不处理账号登录流程，不保存账号密码，不读取或导出用户登录凭证。

插件不得实现以下能力：

- 绕过验证码
- 绕过登录或账号安全校验
- 伪装浏览器指纹或规避平台风控
- 破解、逆向或绕过 Boss 直聘接口限制
- 使用未授权接口批量抓取数据

遇到以下情况必须暂停自动化任务，并在 UI 和日志中说明原因：

- 未登录或登录失效
- 出现验证码、人机校验、安全验证
- 页面结构不符合预期
- 连续失败次数达到阈值
- 达到每日投递上限
- 用户手动暂停
- 当前页面不是 Boss 直聘目标页面

## 3. 第一版范围

第一版实现无人值守批量辅助投递，但必须有保守限速和异常暂停机制。

职位筛选条件：

- 岗位关键词
- 城市
- 薪资范围

投递沟通内容：

- 使用变量模板
- 支持多个模板轮换
- 支持变量：`{jobTitle}`、`{company}`、`{city}`、`{salary}`
- 变量缺失时替换为空字符串，并记录模板警告

默认限速策略：

- 每日最多投递 30 个职位
- 每次操作之间随机等待 60-180 秒
- 连续失败达到 3 次后暂停

第一版不包含：

- AI 自动生成沟通内容
- 后端服务
- 多设备同步
- 账号体系
- 平台接口逆向

## 4. 用户界面需求

插件至少包含弹窗页或设置页，提供以下功能：

- 启动自动投递
- 暂停自动投递
- 查看当前运行状态
- 配置岗位关键词
- 配置城市
- 配置薪资上下限
- 配置每日投递上限
- 配置随机间隔最小值和最大值
- 编辑消息模板
- 查看今日投递进度
- 查看投递日志
- 导入 JSON 配置
- 导出 JSON 配置
- 清空投递日志

UI 需要明确展示风险状态，例如：运行中、已暂停、达到上限、等待人工处理、页面异常。

## 5. 插件架构

建议模块划分：

- React UI：负责配置、状态展示、日志查看、导入导出
- Background service worker：负责全局状态、任务调度、限速、消息转发
- Content script：负责 Boss 页面信息读取、页面状态检测、点击和输入操作
- Shared domain logic：负责筛选规则、薪资解析、模板渲染、限速判断、日志类型定义

Content script 只执行页面内必要操作。核心规则和数据处理逻辑应尽量放在可测试的共享模块中。

## 7. 自动化流程

基础流程：

1. 用户在插件 UI 中配置筛选条件和消息模板。
2. 用户打开 Boss 直聘网页版并保持已登录。
3. 用户点击开始。
4. Background 将任务状态设为运行中。
5. Content script 检测当前页面是否可操作。
6. Content script 读取职位列表或当前职位详情。
7. 插件按关键词、城市、薪资筛选职位。
8. 对符合条件的职位，进入详情或打开沟通入口。
9. 插件选择一条变量模板并渲染消息。
10. 插件填写/发送消息或执行投递动作。
11. 插件记录日志并等待随机间隔。
12. 达到上限或出现异常时暂停。

异常处理：

- 未登录：暂停，提示用户登录后重试。
- 验证码/安全验证：暂停，提示需要人工处理。
- 页面结构变化：暂停，记录选择器失效或关键元素缺失。
- 投递按钮不可用：记录 skipped 或 failed，并继续下一个职位。
- 连续失败：达到阈值后暂停。

## 8. 测试需求

必须提供自动化测试覆盖以下纯逻辑：

- 关键词筛选
- 城市筛选
- 薪资范围解析与匹配
- 模板变量替换
- 每日上限判断
- 随机延迟范围判断
- 连续失败暂停判断

必须提供构建验证：

- TypeScript 类型检查通过
- 浏览器扩展构建成功
- Manifest V3 文件输出合法

建议手动验收：

- 在 Chrome/Edge 扩展管理页面加载构建产物
- 插件 UI 可打开
- 配置可保存并重新读取
- JSON 可导入导出
- 在非 Boss 页面启动时会暂停并提示
- 未登录 Boss 页面会暂停并提示
- 达到每日上限后会暂停

## 9. 验收标准

项目可被视为完成，当且仅当：

- `E:\LLM\pubiln` 中存在完整浏览器插件工程
- 工程可通过终端安装依赖、类型检查、测试和构建
- 构建产物可作为 Chrome/Edge 未打包扩展加载
- UI 支持配置、启停、日志、导入、导出
- 核心规则有单元测试覆盖
- 插件不保存账号密码
- 插件遇到登录、验证码、页面异常和上限时会暂停
- 文档说明如何安装、构建、加载和使用插件

## 10. 后续可选增强

后续版本可以考虑：

- 更细的职位筛选条件，例如经验、学历、公司规模、行业、融资阶段
- 日志搜索和统计
- 多模板权重
- 黑名单公司或关键词
- 白名单岗位关键词
- 更完善的页面适配层
- 在用户明确授权后接入 AI 生成个性化沟通内容
